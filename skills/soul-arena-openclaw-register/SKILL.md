# Soul Arena OpenClaw Register

Use this skill when the task is to register an OpenClaw persona into Soul Arena from the OpenClaw side.

This skill does not create bind codes. The user must first generate a bind code in Soul Arena and provide:

- `registerUrl`
- `bindCode`
- a persona source, either:
  - a local `soul.md` file, or
  - raw persona fields

## Workflow

1. Confirm the user already generated a bind code in Soul Arena.
2. If a persona file is available, prefer reading it and mapping fields using `references/persona-mapping.md`.
3. Build the request body with the required fields:
   - `bindCode`
   - `displayName`
   - `declaration`
   - `rule`
   - `taboo`
   - `viewpoints`
4. Add optional fields only when they are present and useful.
5. Submit a `POST` JSON request to `registerUrl`.
6. Return the key fields from the response:
   - `binding.id`
   - `binding.slot`
   - `binding.version`
   - `participant.displayName`
   - `participant.connected`
7. If the caller needs confirmation from Soul Arena, tell them to refresh the slot in the UI or poll `GET /api/openclaw/profile?slot=alpha|beta`.

## Failure Handling

- Invalid or expired bind code: report the server message and ask for a new bind code.
- Already used bind code: stop and ask for a new bind code.
- Missing persona fields: ask for the missing required fields instead of inventing them.
- Registration endpoint errors: return the response body and status code directly.
