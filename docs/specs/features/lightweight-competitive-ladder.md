# 轻量竞技排位

## 目标
- 把现有 `构筑 -> 开战 -> 回放 -> 历史战报` 升级成 `构筑 -> 排位 -> 结算 -> 下一战`
- 第一版优先服务操作者玩家，不做完整赛季和任务系统
- 用积分、连胜、榜单和建议挑战对象，补上持续游玩的目标感

## 排名规则
- 仅统计真实 `orchestrated` battle，不把经典 demo 纳入排行榜
- 初始积分为 `1000`
- 每场 battle 根据双方当前积分计算增减分
- 排序优先级固定为：积分 > 当前连胜 > 胜率 > 最近状态
- 若同一身份打自己，跳过竞技统计，避免自刷战绩

## 新的数据与接口
- `BattlePackage` / `BattleSummary` 新增 `competition` 结算字段
- 新增 `GET /api/arena/leaderboard`
  - 返回当前榜单与当前焦点选手
- 新增 `GET /api/arena/profile`
  - 支持按 `competitorId` 查询
  - 支持按 `slot=alpha|beta` 解析当前已连接参赛者的竞技档案

## 页面变化
- `/`
  - 首页加入排行榜预览、最近关键对局、竞技目标文案
- `/arena`
  - 为甲乙双方展示积分、排名、战绩、连胜和建议挑战对象
  - 在开战前展示本局风险/收益和冲榜价值
- `/arena/[battleId]`
  - 回放页增加排位结算面板
  - 展示积分变化、排名变化、连胜变化、下克上/终结连胜标签
- `/arena/history`
  - 升级为战绩中心
  - 支持全部 / 胜利 / 失败筛选
- `/arena/leaderboard`
  - 独立排行榜页面

## 取舍
- 当前实现选择“推导式竞技层”
- 竞技统计以 battle 持久化为 source of truth，不新增独立赛季表
- 当前建议挑战对象基于榜单相邻名次和连胜目标生成，暂不引入复杂匹配系统
