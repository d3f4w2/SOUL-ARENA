# Specs README

This folder is the lightweight product and engineering source of truth for `Soul Arena`.

## How To Use
- Keep specs short and current.
- Update specs before changing behavior.
- Record major reversals in `decisions.md`.
- Record shipped work in `implementation-log.md`.

## Core Files
- `product.md`: product goal, audience, MVP narrative
- `architecture.md`: system modules and data flow
- `tech-stack.md`: stack and implementation choices
- `progress.md`: current status only
- `implementation-log.md`: factual change log
- `features/*.md`: active feature specs
- `ui/*.md`: screen map, design rules, key flows
- `api/*.md`: route contracts and env requirements

## Change Order
1. Update the relevant feature spec.
2. Update shared UI/API docs if needed.
3. Implement.
4. Update `implementation-log.md` and `progress.md`.
